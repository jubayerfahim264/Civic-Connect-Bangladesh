# Firebase Configuration Guide for CivicConnect

This guide walks you through setting up Firebase Authentication and Firestore for the CivicConnect application.

## Overview

CivicConnect uses Firebase for:

- **Authentication**: Email/Password and Google Sign-in
- **Database**: Firestore for storing users, services, and subscriptions
- **Cloud Functions**: Automated email notifications to subscribed users

---

## Step 1: Create a Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com/)
2. Click **"Create a project"**
3. Enter project name: `CivicConnect` (or your preferred name)
4. Follow the setup wizard:
   - Disable Google Analytics (optional)
   - Create the project

---

## Step 2: Register Your Web App

1. In the Firebase Console, click the **Web icon** (`</>`)
2. Register a new web app with nickname: `CivicConnect Web`
3. **Copy the Firebase configuration object** that appears—you'll need this in Step 4

---

## Step 3: Enable Firebase Services

### Enable Authentication

1. Go to **Build** → **Authentication**
2. Click **"Get Started"**
3. Enable the following sign-in methods:
   - **Email/Password**: Click, enable, and save
   - **Google**: Click, enable, add your email, and save

### Enable Firestore Database

1. Go to **Build** → **Firestore Database**
2. Click **"Create Database"**
3. Choose region: `us-central1` (or closest to your users)
4. Start in **Test Mode** (for development)
5. Click **"Create"**

---

## Step 4: Add Firebase Configuration to Your Project

1. Open `src/lib/firebase.ts` in your project
2. Replace the placeholder config with your actual Firebase credentials:

```typescript
const firebaseConfig = {
  apiKey: "YOUR_API_KEY", // From Firebase Console
  authDomain: "YOUR_PROJECT.firebaseapp.com",
  projectId: "YOUR_PROJECT_ID",
  storageBucket: "YOUR_PROJECT.appspot.com",
  messagingSenderId: "YOUR_SENDER_ID",
  appId: "YOUR_APP_ID",
};
```

**How to find each value:**

- Go to **Project Settings** (⚙️ icon in Firebase Console)
- Click the **Web App** you registered
- Copy the configuration object under "firebaseConfig"

3. Save the file. Your app will now connect to Firebase.

---

## Step 5: Set Up Google Sign-In

1. In the Firebase Console, go to **Authentication** → **Settings**
2. Copy your **Web Client ID** from the Google provider settings
3. No additional setup needed in the code—it's already configured in `src/lib/firebase.ts`

---

## Step 6: Create Firestore Collections

### users Collection

1. In Firestore, click **"Create collection"**
2. Name: `users`
3. Add a sample document with fields:
   ```
   uid: (auto-generated by auth, but you can set it manually)
   email: "test@example.com"
   displayName: "John Doe"
   photoURL: "https://..."
   subscribedToEmails: true
   createdAt: (timestamp)   lastActive: (timestamp)      # optional: update when user interacts, used for online/offline status and traffic charts   ```

### services Collection

1. Create another collection named `services`
2. Add a sample document:
   ```
   title: "Passport Services"
   category: "Legal Documents"
   description: "Apply for passport..."
   steps: ["Visit office", "Submit forms", "Pay fee"]
   createdAt: (timestamp)
   ```

### messages Collection

1. Create a collection called `messages` (used by the contact form).
2. Documents should include fields:
   ```
   name: string
   email: string
   category: string  // e.g. IT, Politics, Warfare
   message: string
   uid?: string        // optional auth UID when sender is signed-in
   city: string
   country: string
   createdAt: timestamp
   ```
   The client code automatically attaches a server timestamp and, if the
   user is logged in, their Firebase `uid` so they can later read their own
   messages. (Geolocation code was removed to avoid CORS issues.)

## Step 7: Set Up Firestore Security Rules (Important!)

1. Go to **Firestore** → **Rules**
2. Replace the default rules with:

```firestore
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Users can read/write their own user documents
    match /users/{userId} {
      allow read, write: if request.auth.uid == userId;
    }

    // Anyone authenticated can read services
    match /services/{document=**} {
      allow read: if request.auth != null;
    }

    // Only admins can create/edit/delete services
    // (You can add admin role checking here later)
    match /services/{document=**} {
      allow write: if false; // Set to true for admins only
    }

    // Messages created via public contact form
    match /messages/{messageId} {
      // public contact form – anyone may submit a message
      allow create: if true; // or `if request.auth != null` to require login

      // Only the sender (uid stored on the document) or an admin may read,
      // update, or delete the message.  We store the auth UID at create time.
      allow read, update, delete: if request.auth != null && (
        request.auth.token.admin == true ||
        request.auth.uid == resource.data.uid
      );
    }
  }
}
```

3. Click **"Publish"**

## Step 8: Test the Setup

1. Navigate to your app's preview
2. Go to `/signup` and create an account with email
3. Test Google Sign-in from `/login`
4. Go to `/profile` and toggle "Subscribe to Email Updates"
5. Check the Firestore Console to verify the user document was created

## Step 9: Deploy Cloud Functions (Optional)

To enable automated email notifications when new services are added:

1. Install Firebase CLI:

   ```bash
   npm install -g firebase-tools
   ```

2. In your project root, create a `functions` directory:

   ```bash
   firebase init functions
   ```

3. Deploy:
   ```bash
   firebase deploy --only functions
   ```

## Troubleshooting

### "Firebase not initialized" error

- Ensure `src/lib/firebase.ts` has your correct config values
- Check the browser console for specific error messages

### Authentication not working

- Verify Email/Password is enabled in **Authentication** settings
- For Google Sign-in, ensure the app is not on `localhost:3000` (use `localhost:5173` for Vite)

### Firestore write fails

- Check Firestore Security Rules (Step 7)
- Verify the user is authenticated (check `useAuth()` hook)
- Ensure the collection structure matches the rules

### Still having issues?

- Check the [Firebase Documentation](https://firebase.google.com/docs)
- Review console errors in your browser's Developer Tools
- Check Firebase Console → Logs for function errors
